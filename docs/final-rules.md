# 2026 INCOGNITO Final 본선 규칙

## 1. 대회 형식

1. 본 대회는 `Jeopardy + Live Jeopardy` 형식으로 진행됩니다.
2. `Jeopardy`는 대회 시작 시각인 09:30부터 종료 시각인 17:00까지 개방됩니다.
3. `Live Jeopardy`는 시간표에 지정된 특정 시간대마다 팀별 1인이 참여하여 제한 시간 동안 문제 풀이를 진행합니다.
4. `Jeopardy`와 `Live Jeopardy` 모두 추가 문제 개방은 예정되어 있지 않습니다. 단, 심각한 언인텐이 발생한 경우 예외적으로 리벤지 문제를 개방할 수 있습니다.

## 2. 기본 규칙

- 본 대회는 최대 4인의 팀전으로 진행되며, 예선을 통과하고 인증을 완료한 팀만 참가할 수 있습니다.
- 타 팀과 정답, 풀이 방법, 힌트를 어떠한 방식으로도 공유할 수 없습니다.
- 모든 플래그는 `INCOGNITO{***}` 형식이며, 문제를 정상적으로 해결하면 획득할 수 있습니다.
- 동점 발생 시 먼저 점수를 획득한 팀이 더 높은 순위를 차지합니다.
- 대회 진행 중 플래그에 대한 brute-force attack은 금지됩니다.
- 인가된 방법 외의 비정상적인 서버 접근이나 서버 공격 행위는 허용되지 않습니다.
- 심각한 언인텐이 발생한 경우, 운영진 판단에 따라 리벤지 문제를 개방할 수 있습니다.
- 모든 문의는 온라인 Discord 티켓을 통해서만 접수합니다.
- 리스닝 서버가 필요한 경우 Discord 티켓을 통해 운영진에게 문의해야 합니다.
- 점심식사는 배부된 식권을 지참하여 지정된 식사 장소에서 자유롭게 진행합니다.
- 대회장 내부에서는 식사가 불가능합니다.
- 점심시간 종료 10분 전인 12:50까지는 가급적 대회장으로 복귀해 착석해야 하며, Live Jeopardy 참가자는 반드시 복귀해야 합니다.
- 규칙 위반 또는 부정행위 적발 시 운영진 판단에 따라 향후 1년간 대회 참가 자격이 박탈될 수 있으며, 모든 민·형사상 책임은 참가자 본인에게 있습니다.
- 공지와 규칙을 확인하지 않아 발생하는 모든 상황의 책임 역시 참가자 본인에게 있습니다.

## 3. Live Jeopardy 세부 규칙

- 각 팀은 정해진 시간에 1명씩 참여하여 1시간 동안 문제 풀이를 진행합니다.
- 모든 종류의 LLM 및 AI 도구 사용은 금지됩니다.
- Live Jeopardy 참가자는 기존 Jeopardy 풀이에 사용 중이던 LLM을 중지한 뒤 참여해야 합니다.
- 한 번 Live Jeopardy에 참가한 인원은 다른 Live Jeopardy 분야에 중복 참가할 수 없습니다.
- 학교 WiFi(`eduroam` / `guest wifi`)만 사용할 수 있으며, VPN·브릿지 등 우회 수단은 일절 금지됩니다.
- 분야는 `PWN`, `REV`, `WEB`, `Digital Forensics`이며, 각 분야당 1문제가 출제됩니다.
- 본선 참가 팀 인원이 4명 미만인 경우 부족한 인원 수만큼 참여 횟수가 증가합니다.
  - 1인 팀: 1인이 최대 4회 출전 가능
  - 2인 팀: 1인당 최대 2회 출전 가능
  - 3인 팀: 1명만 최대 2회 출전 가능

## 4. 부정행위

### 4.1 정의

- **플래그 공유:** 타 팀과 플래그를 직접 공유하는 행위
- **외부 지원:** 팀원 외 인원이 문제 풀이에 참여하는 행위
- **시스템 공격:** 서버 DDoS, 무차별 대입 공격 등 rate limit을 초과하는 행위
- **방해 행위:** 다른 팀의 문제 풀이를 방해하는 행위
- **중복 참가:** 한 명이 여러 팀에 소속되는 행위
- **backup team 금지:** 등록된 참가 인원 외 인원이 참여하는 행위
- **Live Jeopardy 위반:** 학교 WiFi 외 네트워크 사용, VPN·브릿지 등 우회 사용, local LLM을 포함한 모든 종류의 AI 사용

### 4.2 제재

- **경고:** 의도치 않은 경미한 위반
- **점수 차감:** 해당 문제 점수 무효화
- **즉시 실격:** 고의적 또는 악의적 부정행위
- **영구 제재:** 심각한 서버 공격 시 향후 대회 참가 금지

서버 DDoS, brute force 등은 고의성이 없고 피해가 경미하다고 판단되는 경우를 제외하고, 고의적일 경우 즉시 실격 및 영구 제재 대상이 됩니다.

### 4.3 신고 및 문의

- 현장 운영진을 통한 직접 문의는 받지 않습니다.
- 모든 문의는 Discord 티켓을 통해 접수해야 합니다.
- 운영진 개인 DM으로는 문의를 받지 않습니다.

## 7. Live Jeopardy에서의 LLM 및 AI 도구 사용

### 7.1 기본 원칙

- Live Jeopardy 풀이 시 모든 종류의 LLM 및 AI 도구 사용은 금지됩니다.
- 문제 풀이, 코드 작성, 디버깅, 취약점 분석, exploit 작성 등 문제 해결 과정은 참가자가 직접 수행해야 합니다.
- 일반적인 키워드 기반 웹 검색과 공개 자료 열람은 허용됩니다.

### 7.2 허용되는 행위

- 공식 문서, 기술 블로그, Stack Overflow, GitHub Issue 등 공개 자료 열람
- 일반적인 라이브러리, 프레임워크, 명령어, 에러 문구 검색
- Google 검색 결과에 자동 노출되는 AI Overview 등 의도치 않게 표시된 AI 요약 정보 확인

### 7.3 금지되는 행위

- 문제 설명을 그대로 검색창에 입력하여 AI 요약 답변을 얻는 행위
- 문제 코드, 서버 응답, 에러 로그, 분석 결과 등을 입력하여 AI Overview에 의도적으로 풀이 방향을 얻는 행위
- “이 CTF 문제 어떻게 풀어?”, “이 코드 취약점 찾아줘”, “이 에러로 exploit 작성해줘”처럼 질문형 검색어로 AI 답변을 유도하는 행위
- AI Overview 또는 AI 검색 답변을 생성하기 위해 검색어를 반복 수정하는 행위
- AI Overview에 표시된 풀이 절차, 코드, 디버깅 결과, exploit 작성 방법 등을 참고하는 행위
- Google AI Mode처럼 사용자가 직접 AI 검색 기능을 실행하는 행위

즉, 일반적인 에러 문구 검색은 허용되지만, 문제 맥락을 포함해 AI에게 풀이·디버깅·취약점 분석을 맡기는 방식의 검색은 금지됩니다.

### 7.4 금지되는 도구 및 기능

다음과 같은 도구 사용은 금지됩니다.

- ChatGPT
- Claude
- Gemini
- Copilot
- Cursor AI
- VSCode AI Editor
- JetBrains AI Assistant
- Local LLM
- 기타 AI 기반 코드 작성·분석 도구

또한 운영진이 일반 자동완성과 AI 자동완성을 현장에서 명확히 구분하기 어렵기 때문에, 코드 자동완성·코드 추천·코드 생성 기능은 AI 여부와 관계없이 금지됩니다.

금지되는 기능 예시는 다음과 같습니다.

- 코드 자동완성
- 함수명·변수명 자동완성
- import 자동완성
- IntelliSense / LSP 기반 자동완성
- 코드 추천
- 코드 생성
- 코드 수정
- 리팩터링 제안
- 디버깅 보조
- 취약점 분석 보조
- exploit 작성 보조

단, 코드 내용을 직접 제안하지 않는 단순 편집 기능은 허용됩니다.

### 7.5 화면 공유·캡처 기반 AI 활용 금지

- VMware, 원격 데스크톱, 화면 공유, 캡처 공유 등을 통해 문제 화면이나 풀이 과정을 AI 도구에 전달하여 도움을 받는 행위는 금지됩니다.
- 문제 화면, 코드, 서버 응답, 풀이 과정 등을 촬영하거나 캡처하여 AI 도구에 입력하는 행위 역시 제재 대상이 될 수 있습니다.

### 7.6 정리

- 일반적인 검색과 공개 자료 열람은 허용됩니다.
- 일반적인 에러 문구 검색은 허용됩니다.
- 의도치 않게 노출되는 AI 요약 정보는 허용됩니다.
- AI에게 문제 풀이, 코드 작성, 디버깅, 취약점 분석, exploit 작성을 맡기는 행위는 금지됩니다.
- 코드 자동완성·코드 추천·코드 생성 기능은 AI 여부와 관계없이 금지됩니다.
